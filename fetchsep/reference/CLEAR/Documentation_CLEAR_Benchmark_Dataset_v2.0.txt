Consistent and Reproducible Analysis with FetchSEP
--------------------------------------------------
FetchSEP automatically analyzes SEP enhancements, ensuring consistent and reproducible SEP event definitions with time and across experiments. FetchSEP is composed of three major packages: idsep, opsep, and associations.

Idsep works in an iterative manner to cleanly identify the particle background and enhancements above background that are most likely due to SEP events, using a set of selection criteria. Idsep outputs the mean background value for every point in time for the full GOES datasets and the expected level of variation within the background (sigma). Idsep also provides rough start and end times of SEP events above background (hence the name ID-SEP). Those products were used as a starting point to create the hand-curated "batch_event_list" files for CLEAR.

Opsep calculates SEP event parameters by applying specific SEP event definitions. For the Benchmark dataset, these event definitions include threshold crossings by specific energy channels (e.g., >10 MeV, 10 pfu) AND the identification of SEP events when they increase above the background levels. The mean background and sigma calculated by idsep are used to identify enhancements above background. Opsep always applies SRAG's operational event definitions of >10 MeV exceeds 10 pfu and >100 MeV exceeds 1 pfu when possible (hence the name Op-SEP). Event definitions applied in the CLEAR SEP Benchmark Dataset are:
- >10 MeV exceeds 10 pfu
- >10 MeV exceeds background
- >100 MeV exceeds 1 pfu
- >100 MeV exceeds background
- >30 MeV exceeds 1 pfu
- >30 MeV exceeds background
- > 50 MeV exceeds 1 pfu
- >50 MeV exceeds background

Associations contains modules to extract flare, CME, and solar cycle information from various online catalogs and reference SEP lists stored in the fetchsep repository. These packages are currently able to extract information from NOAA Science X-ray flare summary files, SOHO LASCO CDAW CME catalog, DONKI CME catalog, and solar cycle definitions using the Badhwar-O'Neill (BON) GCR model definitions of solar minimum. 


SEP Lists Used for Flare, CME, etc Associations
-----------------------------------------------
Four SEP lists that were published in the literature or provided via personal communication were used to provide the associated contextual information for the source eruptions (flare, CME, radio, location) and solar wind (shock passages) for each SEP event identified by FetchSEP. The lists were converted to machine-readable formats and then supplemented, whenever possible, with NOAA science X-ray flare values, CDAW CME parameters, and DONKI CME parameters. When needed, SEP Reference Start and End columns were added to give an approximate frame during which the SEP event was likely to be observed in the proton flux as historical lists often provided times that were not specific or not specific to a single satellite. The reference SEP lists were pruned such that each SEP event was represented only once across the lists (filenames contain _unique). 

SRAG SEP list curated by A. Steve Johnson (Leidos, SRAG) which spans Solar Cycle 22 to the present. The list is most complete for larger events that exceed SRAG's operational thresholds, but includes many smaller events that do not. Steve compiled many types of contextual information with exact flare, CME, radio, and SEP values and timing. This list was used as the primary source of associations in the Benchmark dataset. Because this list contains detailed SEP proton information, it is a good standalone SEP list in its own right. 
Machine-readable version:  fetchsep/reference/SRAG_SEP_List.csv

25 MeV SEP list compiled by Ian Richardson from 1967 to the present. The SEPs from 1986 - 1997 were used for the CLEAR dataset. This list was derived from proton measurements by the IMP and SOHO satellites, which have much lower backgrounds compared to GOES, and therefore contains many more small events than the SRAG SEP list. This list contains approximate timing, flare magnitudes, source eruption location information, and CME speeds.  
Machine-readable versions: fetchsep/reference/IGR_25MeV_SEP_List_Full.csv, fetchsep/reference/IGR_25MeV_SEP_List_unique.csv

Cane et al. (2010) list (JOURNAL OF GEOPHYSICAL RESEARCH, VOL. 115, A08101, doi:10.1029/2009JA014848, 2010) spans 1997 to 2006 and contains flare, CME, and Type III radio information along with approximate times of the source eruption and the following proton event and some additional features captured in the comments.
Machine-readable versions: fetchsep/reference/Cane_SEP_List_Full.csv, fetchsep/reference/Cane_SEP_List_unique.csv

Ian Richardson's (2025) SOHO/STEREO list published online at https://dataverse.harvard.edu/dataset.xhtml?persistentId=doi:10.7910/DVN/GQPCXZ. The version used in this work was provided directly by Ian and is consistent with https://link.springer.com/article/10.1007/s11207-026-02614-4. This list contains flare, CME, and radio information and particle intensities at SOHO, STEREO-A, and STEREO-B. 
Machine-readable versions: fetchsep/reference/IGR_SEPlist3_20260306_Full.csv, fetchsep/reference/IGR_SEPlist3_20260306_unique.csv


Transparency and User Deployment
--------------------------------
The CLEAR dataset may be created by any user from scratch using the above version of FetchSEP and using the scripts: 
fetchsep/reference/CLEAR/deploy_CLEAR_Mac.sh
fetchsep/reference/CLEAR/deploy_CLEAR_Linux.sh
fetchsep/reference/CLEAR/deploy_CLEAR_Windows.bat

The scripts refer to the hand-curated time periods in the lists:
fetchsep/reference/CLEAR/batch_event_list_GOES-06_integral_enhance_idsep_CLEAR.txt, etc

Additional files:
fetchsep/reference/CLEAR/fetchsep_CLEAR.cfg: global config parameters. This file is no longer needed as the config values are set in the deploy scripts via command line.
fetchsep/reference/CLEAR/info.txt: Time periods excluded when curating the batch files because of obvious bad data or large data gaps that would make SEP event parameters unreliable.

The _Mac version of the deploy scripts explicitly shows every step of the process as an individual command. It is a good resource to understand the FetchSEP workflow or to run a single spacecraft individually.

Running the deploy script from scratch will:
- download all GOES data from GOES-06 to present in its original format into a data/ folder
- store the original GOES fluxes in simple csv files, always taking the West-facing detector (if there are two)
- calculate the mean background and other products for each complete GOES time series with idsep
- copy batch files from the fetchsep/reference/CLEAR directory
- analyze each SEP event and non-event period independently with opsep

The full process will take 20+ hours, requires a good internet connection, and will result 30 GB+ of data on your computer including the original downloaded GOES files. 

The mean background calculation requires the most time. You may also download the Benchmark Dataset v2.0 from CCMC and use the already-calculated mean backgrounds. To process the individual SEP and non-event periods only, you may run the scripts as:
./fetchsep/reference/CLEAR/deploy_CLEAR_Mac.sh BATCH


Adding New Event Definitions
-----------------------------
You may apply new event definitions (energy channel + threshold) and regenerate your own version of the Benchmark Dataset by modifying the _deploy scripts. You must have the mean backgrounds in the CLEAR/output/idsep folder. To add or change the thresholds, modify the --Threshold tag in:

python bin/fetchsep_automated_pipeline --StartDate 1986-01-01 --EndDate 1994-12-01 --Experiment GOES-06 --FluxType integral --Threshold "30,1;50,1" --BatchFile batch_event_list_GOES-06_integral_enhance_idsep_CLEAR.txt  --IDSEPEnhancement --Associations --StartPoint BATCH --outpath "${outpath}" --plotpath "${plotpath}" --listpath "${listpath}" --opsep_nsigma "${opsep_nsigma}"  > "${outpath}"/GOES-06_integral_batch.log
 

To add a different threshold for an integral channel, for example >10 MeV exceeds 100 pfu:

--Threshold "30,1;50,1;10,100"


To apply a threshold to a differential channel:

python bin/fetchsep_automated_pipeline --StartDate 2010-05-01 --EndDate 2018-01-01 --Experiment GOES-13 --FluxType differential --options "S14;Bruno2017;uncorrected" --Threshold "30,1;50,1" --BatchFile batch_event_list_GOES-13_differential_uncor_S14_B17_bgsub_enhance_idsep_CLEAR.txt --IDSEPEnhancement --IDSEPSubtractBG --Associations --StartPoint BATCH  --outpath "${outpath}" --plotpath "${plotpath}" --listpath "${listpath}" --opsep_nsigma "${opsep_nsigma}"  > "${outpath}"/GOES-13_differential_uncor_S14_B17_bgsub_batch.log


Add a threshold of 0..01 to the energy bin 32.5-56.4 MeV:

--Threshold "30,1;50,1;32.5-56.4,0.01"


Once you have modified the deploy script with your event definition, you may reprocess the SEP parameters and plots by running:
./fetchsep/reference/CLEAR/deploy_CLEAR_Mac.sh BATCH


If you are not interested in enhancements above background, you do not need the idsep files. You must modify the batch_event_list_* files to set IDSEPEnhancement and IDSEPSubtractBG to empty strings. Remove the --ISEPEnhancement and --IDSEPSubtractBG flags from the bin/fetchsep_automated_pipeline calls. Run the script as normal and only the event definitions with static thresholds will be applied.



BENCHMARK DATASET PRODUCTS
==========================

THE CLEAR Benchmark SEP Event Lists
-----------------------------------
GOES_integral_PRIMARY.1986-02-03.2025-09-10_sep_events.csv: Operational SEP event list
GOES_differential_energy_bin_calibrated_PRIMARY.2010-08-02.2017-09-20_sep_events.csv: Energy-bin calibrated SEP event list

For the Operational list, the integral fluxes provided by NOAA were used as-is, retaining the original flux values but setting fluxes identified as background to zero. The Operational list is best used for statistical analysis, development and validation of models with the goal of predicting operational values, and as a historical record of operational values.

For the Energy-bin calibrated list, the uncorrected differential fluxes provided by NOAA were used. The background was subtracted using the idsep mean backgrounds. Calibrated energy channels from the literature (Sandberg et al. 2014, Bruno 2017) were applied and the integral energy channels (>10, >30, >50, >100 MeV) were estimated. The Energy-bin calibrated list is best used for scientific studies, model development and validation for scientific purposes, and dosimetry calculations. The fluence spectra in this list are in calibrated differential energy bins.


See BENCHMARK DATASET PARAMETERS below for a description of each column.

Quality Flags
-------------
Quality flags to indicate that the user may want to take a closer look:
I: Time period ended while SEP was still enhanced (interrupted)
G: A large data gap is present 
T: A data gap likely affects the start or end time of an SEP event 
E: The particle environment was already enhanced at the start of the time period
S: The max flux is likely an erroneous spike in the data


Case
----
Descriptor of event source.
This information is taken primarily from the SRAG SEP list and may not be consistently indicated for all SEP events, with the exception of ESP and Farside. 
The ESP status is indicated in the SRAG list and in some places in the Cane et al. (2010), IGR SOHO/STEREO, and IGR 25 MeV lists. May not be fully complete, but is more complete than in Benchmark Dataset v1.0.
Farside is applied automatically by FetchSEP if event source longitude or DONKI CME longitude is available. All longitudes beyond +/-90 degrees are assigned Farside.

AR: solar eruption source; only included if helps to interpret another case, e.g. AR/ESP.
ESP: peak in lower energies the result of a CME passage or enhancement only due to shock passage 
Farside: solar source beyond the limb
FE: the result of a filament eruption
PrevEv: Previous event ongoing at this event start
Unknown: unknown event source
Pending: source/analysis still to be determined


Flare Location
--------------
For the SRAG list, Event Source location generally refers to the flare location, in order of preference:
- Optical location from SWPC Edited Solar Events product (https://www.swpc.noaa.gov/products/solar-and-geophysical-event-reports) 
- From weekly SWPC Preliminary Report and Forecast of Solar Geophysical Data (PRF) (https://www.swpc.noaa.gov/products/report-and-forecast-solar-and-geophysical-activity) 
- GOES SXI in event list if no optical (During G12)
LMSAL Solar Soft (after Oct 2002) https://www.lmsal.com/solarsoft/latest_events_archive.html 
- If no optical a prorated position calculated from last PRF longitude
- Farside is estimated using solar rotation to highly likely candidate
- No location if no likely candidate stands out
- Pure ESP events no location since enhancement is due to shock not AR

For the Cane et al. list:
- SWPC: H alpha location was obtained by SWPC: Solar Geophysical Data (SGD)
